# Task handoffs

One file per delegated task. The task brief is the contract; its handoff is the durable
execution report and review record.

Implementers must record files changed, exact gates and results, contract deviations,
unresolved issues, and unrelated working-tree changes before yielding. Chat-only reports
are insufficient.

Delegated agents do not commit unless their assignment explicitly authorizes it. The
reviewer/integrator accepts the result, obtains human approval unless pre-authorized,
commits only the task scope, and records the commit here.
