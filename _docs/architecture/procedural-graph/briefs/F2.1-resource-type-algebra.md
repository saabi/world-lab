# Brief — F2.1: generic resource type algebra

**Type:** type-system extension, first milestone of the Foundation 2 generic-resources sequence ·
**Package:** `@world-lab/graph`, `@world-lab/runtime-webgpu` · **Depends on:** Foundation 1,
complete — specifically F1.5's `TypeRef` (`buffer`/`texture`/`sampler` variants already exist as
type-level stubs) and F1.3's `PrimitiveImplementation` (`{kind: 'resource'; descriptor:
ResourceDescriptor}` already reserved as a placeholder) · **Design authority:**
[foundation-2-generic-resources-plan.md](../foundation-2-generic-resources-plan.md) ·
[elemental-webgpu-architecture-review.md, Foundation 2](../elemental-webgpu-architecture-review.md#roadmap-realignment)
· **Contract author:** Opus · **Recommended executor:** Cursor or Codex · **Status:** ready to

## Context

First of Foundation 2's sequence (F2.1 → F2.2 → F2.3 → F2.4 → proof). **This milestone is pure
types and inference — no GPU allocation, no runtime execution.** It gives `ResourceDescriptor`
(currently `{ placeholder?: never }`, `packages/graph/src/implementation.ts:35-37`) its real
shape, and extends F1.5's already-introduced `TypeRef` buffer/texture/sampler variants rather than
inventing a parallel type system.

**Do not reinvent what F1.5 already built.** `TypeRef` already has:
```ts
// packages/graph/src/types.ts, current, unchanged since F1.5
export type BufferAccess = 'read' | 'read-write';
export type BufferUsage = string; // placeholder — this milestone's job
export type TextureDimension = '1d' | '2d' | '3d' | 'cube';
export type SampleType = 'float' | 'depth' | 'sint' | 'uint';
export type StorageTextureAccess = 'read' | 'write' | 'read-write';

type TypeRef =
  | ... // scalar/vector/matrix/array/struct/legacy, all real since F1.5
  | { kind: 'buffer'; element: TypeRef; access: BufferAccess; usages: BufferUsage[] }
  | { kind: 'texture'; dimension: TextureDimension; sample: SampleType;
      format?: string; access?: StorageTextureAccess }
  | { kind: 'sampler'; filtering: boolean; comparison: boolean };
```
This milestone's job is: (1) replace the `BufferUsage = string` placeholder with a real WebGPU
usage-flag union, (2) define `ResourceDescriptor` as a lifetime-qualified wrapper around these
existing `TypeRef` buffer/texture/sampler shapes (not a new, separate resource type algebra), and
(3) write the usage-inference function that derives `usages`/access from how a resource is
actually wired in a graph, rather than requiring every buffer/texture primitive to declare it by
hand.

**Do not confuse this with the pre-existing `ResourceDependency`/`ResourceDataType`**
(`packages/graph/src/types.ts:109-118`) — that's the M8-era external image/mesh/audio resource
declaration mechanism, a document-level input concept, unrelated to GPU buffer/texture allocation.

## Verified current state

- `ResourceDescriptor { placeholder?: never }` and `GpuCommandKind = string`
  (`packages/graph/src/implementation.ts:35-39`) are F1.3's deliberate stubs, unchanged.
- `frameGraph/types.ts`'s `RenderTarget { id, format: GPUTextureFormat, size: TargetSize,
  persistent?: boolean }` (`packages/runtime-webgpu/src/frameGraph/types.ts`) is the texture-only
  precedent this milestone's lifetime model generalizes — `persistent?: boolean` is the existing,
  proven concept for "this resource survives across frames"; F2.1's lifetime classification should
  be a superset of this concept (transient/persistent/history), not a redesign of it.
- `collectFeedbackTargets` (`packages/runtime-webgpu/src/frameGraph/order.ts:139-161`) already
  treats `target.persistent` and `read.previousFrame` as the two signals for "this needs
  ping-pong" — confirmed generic (string IDs and booleans only). F2.1's lifetime model must supply
  values these two signals can continue to key off, generalized to buffers too.
- No usage-inference function exists anywhere today — every current resource-adjacent primitive
  (`buffer.persist`, the pipeline `geometry`/`texture`/`storageBuffer` `DataType` values) declares
  its role by primitive identity, not by inferring usage from graph structure.

## Fix

1. Replace the `BufferUsage = string` placeholder with a real, closed union matching WebGPU's
   actual buffer usage flags:
   ```ts
   export type BufferUsageFlag =
     | 'vertex' | 'index' | 'uniform' | 'storage'
     | 'copy-src' | 'copy-dst' | 'indirect' | 'query-resolve';
   export type BufferUsage = BufferUsageFlag; // keep the exported name stable for existing TypeRef callers
   ```
2. Define resource lifetime as an explicit, shared concept (not per-resource-kind duplicated):
   ```ts
   export type ResourceLifetime =
     | { kind: 'transient' } // scoped to one frame, freed/reused after
     | { kind: 'persistent' } // survives across frames, single physical allocation
     | { kind: 'history'; slots: 2 }; // survives across frames, ping-ponged (current/previous);
                                      // `slots` fixed at 2 for now — N-deep history is out of scope
   ```
   This generalizes `RenderTarget.persistent?: boolean` (`true` → `persistent`, `undefined`/`false`
   → `transient`) and adds the `history` case explicitly, rather than inferring "needs ping-pong"
   implicitly from `previousFrame` reads scattered across passes the way `collectFeedbackTargets`
   currently does at the pass-graph level. F2.2 wires `ResourceLifetime.history` and
   `collectFeedbackTargets`'s existing detection together — this milestone only defines the type.
3. Define `ResourceDescriptor` as a lifetime-qualified wrapper over F1.5's existing buffer/texture/
   sampler `TypeRef` variants:
   ```ts
   export interface ResourceDescriptor {
     id: string;
     shape: Extract<TypeRef, { kind: 'buffer' | 'texture' | 'sampler' }>;
     lifetime: ResourceLifetime;
   }
   ```
   Update `PrimitiveImplementation`'s `{ kind: 'resource'; descriptor: ResourceDescriptor }`
   variant (`packages/graph/src/implementation.ts`) to use this real shape — no change to the
   variant's own field name or position in the union.
4. Write usage inference — given a graph and a resource node, derive `BufferUsage[]`/
   `StorageTextureAccess` from how the resource is actually wired, rather than requiring the
   author to declare it:
   ```ts
   export function inferBufferUsage(doc: GraphDocument, resourceNodeId: string): BufferUsageFlag[];
   ```
   Minimum required inference rules for this milestone: a resource written by exactly one pass and
   read by zero or more others infers `'storage'` (write access) plus `'storage'` (read access) on
   the reader side — do not attempt full usage-flag inference for every WebGPU binding scenario
   (e.g. vertex/index buffer usage inference from a kernel's actual binding layout is Foundation
   3's concern, once kernels exist to bind against). Keep this deliberately minimal and correct
   for the buffer-feedback and storage-read/write cases Foundation 2's own proof needs, not
   exhaustive for every possible WebGPU usage combination.
5. Do **not** implement any GPU allocation, bind-group derivation, or execution here — this
   milestone is types and inference only. `ResourceDescriptor` values are produced and validated,
   never realized. Realization is F2.2 (planning) and F2.3 (allocation)'s job.

## Gate

1. A test proves `ResourceDescriptor`'s `shape` field only accepts `TypeRef`'s `buffer`/`texture`/
   `sampler` kinds — a `scalar`/`vector`/etc. `TypeRef` must not type-check as a valid `shape`
   (`Extract<...>` should make this a compile-time guarantee; write a test that would fail to
   compile if this narrowing were removed, matching this project's exhaustiveness-test convention).
2. A test proves `ResourceLifetime`'s three variants round-trip through serialization unchanged
   (no information loss for `history`'s `slots` field specifically).
3. A test proves `inferBufferUsage` correctly derives `['storage']` (or the appropriate read/write
   split) for a minimal graph: one resource node written by one pass, read by a second pass — the
   concrete case Foundation 2's own buffer-feedback proof sample needs.
4. A test proves `inferBufferUsage` returns an empty or minimal-safe result (not a thrown error)
   for a resource with no writers — deliberately not attempting exhaustive inference for
   incomplete/malformed graphs, matching this milestone's stated minimal scope.
5. `check` **and** `test** green for `graph`, `runtime-webgpu`, full workspace (clear every
   `packages/*/dist` first, per `AGENTS.md`'s documented stale-dist issue).

## Out of scope

GPU resource allocation (F2.3). Bind-group layout derivation (F2.3). Wiring `ResourceLifetime.history`
into `collectFeedbackTargets`'s actual detection logic (F2.2). Full WebGPU usage-flag inference for
every binding scenario (partially Foundation 3's concern, once kernels exist). N-deep (>2 slot)
history. Any change to Foundation 1's milestones or to this document's own Foundation 2 sequencing
beyond this one milestone.

## Handoff

→ `ResourceDescriptor` has a real, lifetime-qualified shape built directly on F1.5's existing
`TypeRef` buffer/texture/sampler variants, with a minimal, correct usage-inference function
proven against the buffer-read/write case Foundation 2's own proof needs. Once this lands: F2.2
(resource dependency planner — generalizing `frameGraph/types.ts` to this milestone's `Resource`
shape, updating `resolveTargetSizes` for resource-kind-aware sizing) is next — request its own
contract at that point.
