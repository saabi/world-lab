## Context

Fourth of five milestones (F1.1 → F1.2 → F1.5 → **F1.3** → F1.4a). **This milestone is now frozen
and approved for implementation** — do not revisit F1.1/F1.2/F1.5 (done) or F1.4a's design here;
F1.4a gets its own contract when it becomes next.

`NodePrimitive.wgsl: WgslSourceRef` is currently a **required** field, forcing every primitive —
including ones with no real WGSL function — to fake one. Five nodes (`buffer.persist`,
`stage.fragment`, `target.display`, `target.mesh`, `geometry.fullscreenPlane`) register a
placeholder module whose entire body is a comment (`packages/procedural-wgsl/src/modules/pipeline/structural.ts`'s
own header: *"honest structural markers, no empty fn stubs"*). Five more primitives
(`host.fragCoord`, `host.iTime`, `host.iResolution`, `procedural.uv`, `procedural.metricPosition`)
register a genuinely fake WGSL module (e.g. `fn frag_coord() -> vec2f { return vec2f(0.0); }`) and
are *also* special-cased by literal primitive-id string checks in `emitGraphEval.ts`. This
milestone introduces a discriminated `PrimitiveImplementation` union so every primitive declares
what kind of thing it actually is, retiring both fakery patterns — without creating a package
dependency cycle between `graph` and `procedural-wgsl` for groups.

**One sequencing issue found and fixed in this contract pass, not present in the strategic plan's
literal text:** the plan's own F1.3 test gate (item 7) asks for a test proving
`preview.fieldSink`/`legacy.consumerSink` declare a real TypeBox `params` schema — but those two
primitives don't exist yet; they're **F1.4a's** job to create via document migration, not F1.3's.
Verified directly: neither name appears anywhere in the codebase today. This milestone proves the
underlying mechanism (a compatibility-sink kind whose per-node data lives in `node.params`, not in
static `SinkDefinition`) with a **test-only** primitive instead — see Fix step 3 and Gate item 7,
revised from the plan's literal wording accordingly. F1.4a will create the real
`preview.fieldSink`/`legacy.consumerSink` using this same mechanism once it's proven here.

## Fix

1. Add the discriminated union to `packages/graph/src/types.ts` (or a new `implementation.ts`,
   whichever this package's convention favors — check how `TypeRef`/`coercion.ts` were organized
   in F1.5 and match it):
   ```ts
   type PrimitiveImplementation =
     | { kind: 'wgsl-function'; moduleId: string; entry: string }
     | { kind: 'group'; groupId: string }
     | { kind: 'host-input'; binding: HostBinding }
     | { kind: 'legacy-structural'; marker: string }
     | { kind: 'sink'; sink: SinkDefinition }
     | { kind: 'resource'; descriptor: ResourceDescriptor }
     | { kind: 'kernel'; stage: 'vertex' | 'fragment' | 'compute' }
     | { kind: 'command'; command: GpuCommandKind };
   ```
   `ResourceDescriptor`/`GpuCommandKind` are placeholders (Foundations 2–4 define their real
   shapes) — a minimal type-level stub is enough (e.g. `interface ResourceDescriptor {
   placeholder?: never }` / `type GpuCommandKind = string`), matching how F1.5 stubbed `TypeRef`'s
   `array`/`buffer`/`texture`/etc. variants. Only `wgsl-function`, `group`, `host-input`,
   `legacy-structural`, and `sink` need real, working definitions this milestone.
2. Define `HostBinding` generically (not a closed ShaderToy-specific enum):
   ```ts
   type ShaderStage = 'vertex' | 'fragment' | 'compute'; // distinct from legacy PipelineStage,
                                                          // which also has 'mesh-gen' — that
                                                          // conflation is F1.4a's to retire
   interface HostBinding {
     context: 'invocation' | 'stage-builtin' | 'playback' | 'write-target' | 'read-resource' |
       'interaction' | 'session';
     key: string;
     stages?: ShaderStage[];
   }
   ```
   Migrate the five host/procedural primitives to `kind: 'host-input'` with standard-library-
   registered `HostBinding` values (e.g. `{context: 'stage-builtin', key: 'fragCoord',
   stages: ['fragment']}` for `host.fragCoord`, `{context: 'playback', key: 'iTime'}` for
   `host.iTime`). Update `emitGraphEval.ts`'s five string-check branches (lines 239, 248, 259, 270,
   279) to dispatch on `implementation.kind === 'host-input'` + `binding.context`/`binding.key`
   instead. Remove their placeholder WGSL modules once nothing reads `.wgsl` for them.
3. Define `SinkDefinition`/`SinkInvocation`:
   ```ts
   interface SinkInvocation<T = unknown> {
     sinkKind: string;
     nodeId: string;
     dependencies: PortRef[]; // generic — resolvable without interpreting `payload`
     payload: T;
   }
   interface SinkDefinition {
     kind: string;
     deriveInvocation(doc: GraphDocument, node: Node): SinkInvocation | null;
   }
   ```
   Migrate `target.display` to `kind: 'sink'`, `deriveInvocation` porting
   `derivePipelinePresentations`'s per-node logic (`packages/graph/src/pipeline.ts:77`) — ported,
   not reimplemented — with `dependencies` populated from the real incoming edge. Migrate
   `target.mesh` the same way, porting `deriveMeshTargets`/`isMeshTarget`
   (`packages/graph/src/meshTarget.ts`) into its `deriveInvocation`, `dependencies` populated from
   the wired position/normal edges.

   **Prove the compatibility-sink mechanism with a test-only primitive, not the real
   `preview.fieldSink`/`legacy.consumerSink` (those are F1.4a's to create):** register a
   test-scoped primitive (e.g. `test.compatibilitySink`) whose `SinkDefinition` declares a real
   TypeBox `params` schema (e.g. `Type.Object({ outputName: Type.String() })`) and whose
   `deriveInvocation` resolves `node.params.outputName` against `doc.outputs` by name — populating
   `dependencies` with the *resolved port*, not from any incoming edge (there is none; this sink
   has no input ports at all). This proves: (a) per-node sink data survives in `node.params`
   through primitive-swap resync (`packages/graph-editor/src/irAdapter.ts:79`'s
   `replaceNodePrimitive`, which only preserves keys declared in the primitive's TypeBox schema —
   confirmed at line 74's `paramKeys`), and (b) `dependencies` correctly comes from name-resolution
   for this sink kind, not edge-walking — exactly the generic mechanism F1.4a will reuse for the
   real compatibility sinks.
4. Define the sink-handler registration contract:
   ```ts
   interface SinkCompilerAdapter {
     sinkKind: string;
     toConsumerDescriptor(invocation: SinkInvocation): ProceduralConsumer;
   }
   interface SinkHandlerRegistry<TAdapter> {
     register(adapter: TAdapter): void; // throws on duplicate sinkKind
     get(sinkKind: string): TAdapter | undefined;
   }
   ```
   `SinkCompilerAdapter`s live in `packages/compiler` (compile-time: turn an invocation into a
   descriptor). Runtime execution handlers (`runtime-webgpu`, actually drawing/dispatching) are a
   *separate* registry — do not conflate compiling a sink with executing it under one mechanism.
5. Give `buffer.persist`, `stage.fragment`, `geometry.fullscreenPlane` (all three, in
   `structural.ts`) `kind: 'legacy-structural'` with a `marker` string (e.g. the primitive id) —
   not `sink` or `command`, since none of them present/export (not sinks) or draw/dispatch (not
   commands), and asserting either now would assign false final semantics ahead of Foundation
   2/3's real resource/pass model.
6. Add `GroupResolver` to `packages/graph`, mirroring `WgslModuleResolver` exactly:
   ```ts
   interface GroupResolver {
     resolve(groupId: string): Promise<GroupDefinition>;
   }
   ```
   Implement `createStandardLibraryGroupResolver` in `packages/procedural-wgsl`, populated from
   compiled group values (e.g. `TRANSFORM_SPHERIFY_GROUP`) already there. Inject it wherever
   `WgslModuleResolver` already is (`compiler`, `graph-editor`, `runtime-webgpu` call sites),
   matching that existing injection pattern exactly. Change `{kind: 'group'; groupId: string}` —
   never embed a `GroupDefinition` value directly in `implementation`, which would force
   `packages/graph` to import compiled values from `packages/procedural-wgsl`, reversing the
   existing (and correct) dependency direction.
7. Add `implementation: PrimitiveImplementation` per the optional-`wgsl`-pair pattern in the
   Verified-state note above, composing with F1.5's existing `normalizePrimitiveInput` (in
   `packages/graph/src/registry.ts`) rather than adding a fourth independent normalization pass.
8. Migrate `codegen.ts:33`, `groupCodegen.ts`, `emitGraphEval.ts` to branch on
   `implementation.kind` instead of assuming `.wgsl` always exists.

## Gate

1. Every existing `wgsl-function` and `group`-backed primitive compiles to byte-identical WGSL
   before and after.
2. `buffer.persist`/`stage.fragment`/`geometry.fullscreenPlane` carry `kind: 'legacy-structural'`
   and no longer have a registered WGSL module; `target.display`/`target.mesh` carry `kind: 'sink'`
   with a working `SinkDefinition.deriveInvocation` reproducing today's
   `derivePipelinePresentations`/`deriveMeshTargets` output exactly (including `dependencies`
   matching the real wired edges); `stage.vertex` keeps its real WGSL module, unaffected; the five
   `host-input` primitives no longer have a placeholder WGSL module and no longer require a
   primitive-id string check in `emitGraphEval.ts`.
3. A test proves the compiler's dispatch skips non-function/non-group kinds without attempting to
   link/resolve a WGSL module for them.
4. A test proves the group registry resolves `groupId` correctly for the editor's "expand group"
   path without `packages/graph` importing `packages/procedural-wgsl` — verify via a
   dependency-direction assertion (e.g. asserting `graph`'s `package.json` has no
   `procedural-wgsl` dependency), not just runtime behavior.
5. A test proves `SinkDefinition.deriveInvocation` for `target.mesh` produces identical
   `MeshTargetDescriptor`-equivalent output to today's `deriveMeshTargets` on every bundled mesh
   sample.
6. A test proves `deriveInvocation` always populates `SinkInvocation.dependencies` — from a real
   incoming edge for `target.display`/`target.mesh`, from resolved `node.params`-based name lookup
   for the test-only compatibility-sink primitive — and that generic code (a hypothetical slicing
   consumer) only ever reads `dependencies`, never `payload`.
7. **Revised from the plan's literal wording, since the real primitives don't exist yet:** a test
   proves the test-only `test.compatibilitySink` primitive's `params` schema data survives
   `replaceNodePrimitive`-style resync (`irAdapter.ts`) across a primitive swap, and that its
   `deriveInvocation` correctly resolves its `node.params`-referenced output name into
   `dependencies` — the mechanism F1.4a will reuse for the real `preview.fieldSink`/
   `legacy.consumerSink`, proven generically here without depending on F1.4a's own deliverables.
8. A test proves `SinkHandlerRegistry.register` throws on a duplicate `sinkKind`, matching
   `registerPrimitive`'s existing collision behavior, and that a `SinkCompilerAdapter` registration
   is independent of any runtime-webgpu execution-handler registration (a separate registry
   instance/type, not shared state).
9. A test proves `implementation`/`wgsl` normalize consistently through the same shared
   `normalizePrimitiveInput` used for `type`/`dataType` and `semantics` — e.g. registering with
   only `wgsl` derives the correct `implementation: {kind: 'wgsl-function', ...}`, and vice versa.
10. `check` **and** `test` green for `graph`, `compiler`, `procedural-wgsl`, `runtime-webgpu`, full
    workspace (clear every `packages/*/dist` first, per `AGENTS.md`'s documented stale-dist issue).
