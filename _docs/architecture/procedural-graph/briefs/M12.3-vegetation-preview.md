## Objective

Create a scene-free 3D preview component in `@virtual-planet/graph-editor` that displays an instanced visualization of the generated vegetation candidates. To simulate level-of-detail (LOD) transitions as a viewer approaches the surface, the preview shifts rendering modes based on a **Simulated Altitude** slider:

```
Simulated Altitude
  │
  ├── ≥ 2000m  ──────► None: flat terrain patch, no vegetation.
  │
  ├── 500m–1999m ────► Statistical: patch colored as a continuous density heatmap.
  │
  ├── 150m–499m  ────► Impostor: patch + instanced 2D quads (billboards) at candidate positions.
  │
  └── < 150m   ──────► Full: patch + instanced 3D cones (representing trees) at candidate positions.
```

## Behavior & Rendering Modes

The renderer configures the canvas with the preferred texture format and draws the scene from a static diagonal looking-down perspective matrix:
- **Target (LookAt center):** Center of the patch: `[patch.widthMeters / 2, 0, patch.heightMeters / 2]`.
- **Eye (Camera position):** Offset from target: `target + [1.5 * patch.widthMeters, patch.widthMeters * 0.8, 1.5 * patch.heightMeters]`.
- **Up vector:** `[0, 1, 0]`.

### 1. `none` Mode (`altitudeMeters >= 2000`)
- Renders only the patch as a flat, neutral-colored plane mesh (e.g. `#1c2130`).
- No compute shader dispatch is performed.
- Returns zero candidates.

### 2. `statistical` Mode (`500 <= altitudeMeters < 2000`)
- Renders a colored density heatmap texture onto the plane mesh.
- Shader compiles the `density` graph output and evaluates it per-pixel in a fragment shader, coloring the surface based on the selected density channel (e.g. green scale for vegetation density).
- Returns zero candidates.

### 3. `impostor` Mode (`150 <= altitudeMeters < 500`)
- Dispatches the compute shader (`executeVegetationCandidateCompute`) to generate candidate instances.
- Renders the plane mesh as a flat base.
- Renders each candidate instance as a **2D vertical quad** (impostor billboard) of width 0.3 and height 0.6. The quad is colored green and faces the camera.
- The instance buffer supplies `candidate.position` to offset the vertex template.

### 4. `full` Mode (`altitudeMeters < 150`)
- Dispatches the compute shader to generate candidate instances.
- Renders the plane mesh as a flat base.
- Renders each candidate instance as a **3D cone** (representing tree geometry, with 5 base vertices on a XZ circle of radius 0.15, and 1 tip vertex at `[0, 0.6, 0]`).
- Scale of the cone templates is driven by candidate `vigor`: `scale = 0.5 + vigor * 0.5`.
- Cones are shaded with flat lighting (using normals, e.g. `#2e7d32` colored).

### `VegetationPreviewPanel.svelte` Layout
- Renders a `<canvas>` for WebGPU rendering.
- Renders selects to choose which graph output represents **Density** (must be a `vec3f` output) and **Placement** (must be a `f32` output). If none are selected or available, displays a helpful validation message.
- Renders a slider for **Simulated Altitude** (`10m` to `2500m`, default `100m`).
- Displays a status log:
  - **LOD Mode:** `None | Statistical | Impostor | Full`
  - **Candidates:** `X / Max` (shows count and overflow state).

## Gate & Validation

### Automated Headless Tests
1. **`vegetationPreview.test.ts`**
   - Validates that invalid config parameters (negative spacing, etc.) throw `RangeError` before reaching WebGPU.
   - Verifies correct mode mapping based on `altitudeMeters` threshold boundaries.

2. **`sceneFree.test.ts`**
   - Must remain green (no forbidden three.js or canvas-scene dependencies imported into `graph-editor` source).

### Manual Verification
- Access `/graph-editor` and choose the **Vegetation** preview tab.
- Drag the **Simulated Altitude** slider and verify the renderer dynamically changes modes:
  - At `2200m`: Neutral grey plane.
  - At `1000m`: Colored density heatmap texture.
  - At `350m`: Flat green billboards standing on the plane.
  - At `50m`: 3D instanced cones standing on the plane.

## Out of Scope
- Actual asset meshes, leaves, branches, or high-fidelity species textures.
- Real-time flight camera movement in this panel (camera remains static lookup).
- Cross-patch suppression or biome mapping.
- Integration into the planet rendering pipeline (that belongs to M13).
