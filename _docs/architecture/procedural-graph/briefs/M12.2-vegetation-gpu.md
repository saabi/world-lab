# Brief — M12.2: GPU vegetation candidate compute (contract)

**Milestone:** M12.2 ([M12-vegetation.md](./M12-vegetation.md)) ·
**Package:** `@virtual-planet/runtime-webgpu` (+ small `@virtual-planet/graph`
primitive for metric position) · **Depends on:** M12.1 ✅ (`bf999aa`), M10.2 ✅
(module resolver + compute consumer pattern) · **Design authority:**
[vegetation.md](../vegetation.md), [inputs-cpu-and-resources.md](../inputs-cpu-and-resources.md) ·
**Contract author:** Opus · **Recommended executor:** Sonnet (GPU compute).

## Objective

Add a **scene-free** WebGPU compute consumer that evaluates the **same**
metric-space peak rules as M12.1 (`generateVegetationCandidates`) into a
bounded storage buffer. Density and placement fields come from **sliced graph
outputs** compiled through the standard module resolver — no vegetation-specific
Graph IR types.

M12.2 is **implementation-ready** when this brief lands. Do not start M12.3
(editor preview) until M12.2 gate is green.

```
VegetationPatch + config + (density graph, placement graph)
  → slice + generateWgsl (×2) + emitGraph*Eval
  → vegetation compute shader (grid + peak logic)
  → storage buffer readback → VegetationCandidateGpuRecord[]
```

## Prerequisites (graph package — small, blocking)

point.

### Types (`vegetationTypes.ts`)

Shapes **must match** M12.1 (`packages/runtime-cpu/src/vegetation.ts`) field
names and validation rules. Duplicate types here (do not depend on `runtime-cpu`
at runtime — keeps `runtime-webgpu` scene-free and acyclic).

```ts
export type Density3 = readonly [number, number, number];
export type VegetationChannel = 0 | 1 | 2;

export interface VegetationPatch {
	id: string;
	origin: readonly [number, number, number];
	tangentX: readonly [number, number, number];
	tangentY: readonly [number, number, number];
	widthMeters: number;
	heightMeters: number;
}

export interface VegetationCandidateConfig {
	spacingMeters: number;
	channel: VegetationChannel;
	placementThreshold: number;
	densityThreshold: number;
	minProminence: number;
	minAltitudeMeters?: number;
	maxAltitudeMeters?: number;
	maxSlope?: number;
}

/** One emitted instance; omits string ids (reconstructed on CPU readback). */
export interface VegetationCandidateGpuRecord {
	ix: number;
	iy: number;
	channel: VegetationChannel;
	position: readonly [number, number, number];
	localMeters: readonly [number, number];
	density: Density3;
	placement: number;
	prominence: number;
	vigor: number;
}

export interface VegetationGraphBinding {
	graph: GraphDocument;
	output: PortRef;
}

export interface VegetationCandidateComputeInput {
	device: GPUDevice;
	patch: VegetationPatch;
	config: VegetationCandidateConfig;
	density: VegetationGraphBinding;   // vec3f output
	placement: VegetationGraphBinding;   // f32 output
	/** Hard cap on records written; see overflow semantics. */
	maxCandidates: number;
}

export interface VegetationCandidateComputeResult {
	patchId: string;
	gridWidth: number;
	gridHeight: number;
	candidateCount: number;
	overflowed: boolean;
	candidates: VegetationCandidateGpuRecord[];
}

export function computeVegetationGridSize(
	patch: VegetationPatch,
	spacingMeters: number
): { gridWidth: number; gridHeight: number };

export async function executeVegetationCandidateCompute(
	input: VegetationCandidateComputeInput
): Promise<VegetationCandidateComputeResult>;
```

`computeVegetationGridSize` uses the **same loop bounds** as M12.1:

```text
ix = 0, 1, … while (ix + 0.5) * spacing < widthMeters
iy = 0, 1, … while (iy + 0.5) * spacing < heightMeters
```

## Metric patch / grid inputs (uniforms)

Single uniform struct `VegetationPatchParams` (std140), written by host before
dispatch:

| Field | Type | Notes |
|-------|------|-------|
| `origin` | `vec3<f32>` + pad | patch minimum corner |
| `tangent_x` | `vec3<f32>` + pad | unit |
| `tangent_y` | `vec3<f32>` + pad | unit, orthogonal |
| `width_meters` | `f32` | |
| `height_meters` | `f32` | |
| `spacing_meters` | `f32` | |
| `grid_width` | `u32` | from `computeVegetationGridSize` |
| `grid_height` | `u32` | |
| `channel` | `u32` | 0–2 |
| `placement_threshold` | `f32` | |
| `density_threshold` | `f32` | |
| `min_prominence` | `f32` | |
| `min_altitude_meters` | `f32` | `NaN` = disabled |
| `max_altitude_meters` | `f32` | `NaN` = disabled |
| `max_slope` | `f32` | `NaN` = disabled |
| `max_candidates` | `u32` | copy of input cap |

Validation on CPU **before** GPU dispatch: same `RangeError` rules as M12.1
(`validatePatch` / `validateConfig` semantics). Invalid input never reaches the
device.

**Position per thread** (ix, iy from `global_invocation_id`):

```wgsl
let x = (f32(ix) + 0.5) * params.spacing_meters;
let y = (f32(iy) + 0.5) * params.spacing_meters;
if (x >= params.width_meters || y >= params.height_meters) { return; }
let position = params.origin + params.tangent_x * x + params.tangent_y * y;
```

Neighbor placement samples use `x ± spacing`, `y ± spacing` with the **same**
world-space formula (may extend outside the patch rectangle — matches M12.1).

## Density / placement graph bindings

### Slicing + codegen

For each binding:

1. Resolve output name from `graph.outputs`.
2. `sliceGraph(graph, { outputs: [name] })`.
3. `generateWgsl(slice, createStandardLibraryResolver())` — merge both modules
   into one shader translation unit (dedupe by module id; fail on collision).
4. Emit evaluate body:
   - density: `emitGraphVec3Eval(density.graph, density.output, { positionExpr: 'position' })`
   - placement: `emitGraphScalarEval(placement.graph, placement.output, { positionExpr: 'position' })`

Merged shader defines:

```wgsl
fn evaluate_density(position: vec3<f32>) -> vec3<f32> { … }
fn evaluate_placement(position: vec3<f32>) -> f32 { … }
```

Graph uniform structs are merged into one `GraphParams` block (same pattern as
`planeScalarPreview` `packParams`); duplicate param fields across graphs are
forbidden — `RangeError` at host side if collisions occur.

### Bind group layout (group 0)

| Binding | Resource | Access |
|---------|----------|--------|
| 0 | `VegetationPatchParams` uniform | read |
| 1 | `GraphParams` uniform | read |
| 2 | `VegetationResultMeta` storage | read_write |
| 3 | `candidates[]` storage array | read_write |

`@workgroup_size(8, 8, 1)` — dispatch
`(ceil(gridWidth/8), ceil(gridHeight/8), 1)`.

## Candidate storage-buffer layout

### `VegetationCandidateGpu` (storage array element)

WGSL struct, **16-byte aligned**, 64 bytes per record:

| Offset | Field | Type |
|--------|-------|------|
| 0 | `ix` | `u32` |
| 4 | `iy` | `u32` |
| 8 | `channel` | `u32` |
| 12 | `_pad0` | `u32` |
| 16 | `position` | `vec3<f32>` |
| 28 | `_pad1` | `f32` |
| 32 | `local_meters` | `vec2<f32>` |
| 40 | `density` | `vec3<f32>` |
| 52 | `placement` | `f32` |
| 56 | `prominence` | `f32` |
| 60 | `vigor` | `f32` |

`vegetationBuffer.ts` exports:

```ts
export const VEGETATION_CANDIDATE_STRIDE = 64;
export function vegetationCandidateBufferByteLength(maxCandidates: number): number;
export function decodeVegetationCandidates(
	data: ArrayBuffer,
	count: number
): VegetationCandidateGpuRecord[];
```

### `VegetationResultMeta` (first 16 bytes of binding 2)

| Field | Type | Notes |
|-------|------|-------|
| `candidate_count` | `atomic<u32>` | incremented per emitted candidate |
| `overflowed` | `atomic<u32>` | set to 1 when a write is skipped due to cap |
| `_pad0` | `u32` | |
| `_pad1` | `u32` | |

After dispatch: host copies meta to a readback buffer, maps `candidate_count` and
`overflowed`, then maps only the first `candidate_count` records from binding 3.

## Capacity and overflow behavior

- **`maxCandidates`** is required on input (no implicit default). Callers may set
  `maxCandidates = gridWidth * gridHeight` for an upper bound.
- Emission order is **grid order** (`iy` outer, `ix` inner) — same as M12.1.
- When `candidate_count` would exceed `maxCandidates`, the shader **does not
  write** the record, sets `overflowed = 1`, and continues iteration (no early
  exit — keeps GPU divergence bounded and deterministic meta).
- Host returns `overflowed: true` and **only** the first `maxCandidates`
  records in grid order.
- `candidateCount` in the result is `min(emitted, maxCandidates)`.

Buffer allocation: `vegetationCandidateBufferByteLength(maxCandidates)` using
`alignTo` from `buffers.ts`.

## Peak algorithm (GPU)

In-shader logic must match M12.1 exactly:

1. Sample placement at center and four metric neighbors at ±spacing on tangent
   axes.
2. Center **strictly greater** than all four neighbors.
3. `prominence = center - max(neighbors)`; require `>= min_prominence`.
4. `center >= placement_threshold`.
5. Sample density; require `density[channel] >= density_threshold`.
6. Optional altitude/slope: **out of scope for M12.2 gate** when fields would
   require a terrain graph. Uniform `min_altitude_meters` / `max_altitude_meters`
   / `max_slope` are wired as `NaN` in the parity fixture and documented as
   reserved for a follow-on (`terrain` graph binding). Host rejects non-finite
   altitude/slope config the same as M12.1 when those fields are set.
7. `vigor` formula matches M12.1 (`placementVigor * density[channel]`, clamped).

Record `ix`, `iy`, `channel`, `position`, `local_meters = (x,y)`, `density`,
`placement`, `prominence`, `vigor`.

## CPU / GPU parity fixture

`fixtures/vegetationParity.ts` exports:

- `PARITY_PATCH` — same geometry as M12.1 test `PATCH` (`3×3` m, spacing `1`).
- `PARITY_CONFIG` — same as M12.1 test `CONFIG`.
- `PARITY_DENSITY_GRAPH` — single `procedural.metricPosition` → stub node
  outputting constant `[0.8, 0.2, 0.1]` via test module
  `test.vegetationParityDensity`.
- `PARITY_PLACEMENT_GRAPH` — `procedural.metricPosition` →
  `test.vegetationParityPlacement` implementing the `twoPeakSamplers()` placement
  field from `vegetation.test.ts` (peaks at `(0.5,0.5)` strength `1` and
  `(2.5,1.5)` strength `0.9`, else `0`).
- `parityResolver()` — `createStandardLibraryResolver()` plus the two test
  modules above (test-only WGSL sources live in `vegetationParity.ts`).

**Parity gate** (`vegetationCandidates.test.ts`):

```ts
import { generateVegetationCandidates } from '@virtual-planet/runtime-cpu';

const cpu = generateVegetationCandidates(PARITY_PATCH, PARITY_CONFIG, cpuSamplersFromParityModules());
const gpu = await executeVegetationCandidateCompute({ …, maxCandidates: 16, … });

expect(gpu.overflowed).toBe(false);
expect(gpu.candidateCount).toBe(2);
expect(recordsEqualModuloId(gpu.candidates, cpu)).toBe(true);
```

`recordsEqualModuloId` compares all fields except string `id` / `patchId`; maps
GPU `(ix,iy,channel)` to CPU id `"${patch.id}:${ix}:${iy}:${channel}"`.

Tolerance: **exact** `f32` equality for parity fixture (deterministic analytic
fields). No epsilon unless a test documents a specific WGSL vs JS divergence.

## Device-unavailable skip behavior

Mirror `consumers/planeScalarPreview.test.ts`:

```ts
const hasWebGPU =
	typeof globalThis.navigator !== 'undefined' &&
	'gpu' in globalThis.navigator &&
	globalThis.navigator.gpu !== undefined;
```

- Layout / codegen / `computeVegetationGridSize` tests run **without** GPU.
- Integration tests use `it.skipIf(!hasWebGPU)(…)`.
- When skipped, vitest reports them as skipped (not failed).
- Do not call `requestGpuDevice()` at module load time.

Expected gate count delta: **runtime-webgpu** gains ≥6 passing headless tests and
≥2 GPU tests (parity + empty plateau), with GPU tests skipped in CI without
WebGPU.
